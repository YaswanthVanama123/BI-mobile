# EnviroMaster BI — Mobile (React Native CLI)

Native mobile companion to the EnviroMaster BI web app. Same screens, same data, same API —
bare **React Native CLI** (no Expo), a professional chart UI, and a feature-based architecture.

## Stack
- **React Native 0.74 (CLI / bare)** — no Expo
- **React Navigation** drawer (grouped sections mirroring the web sidebar)
- **react-native-gifted-charts** (+ react-native-svg) — bar / stacked-bar / line / donut
- **axios** → same `/api/v1` backend as the web app
- **react-native-vector-icons** (Ionicons), **react-native-document-picker** (payroll CSV upload)
- **AsyncStorage** for the auth token
- `@/*` path alias → `src/*` (via `babel-plugin-module-resolver`)

## First-time setup

This repo holds the **JS/config** for the app. The native `android/` and `ios/` folders are
generated by the RN CLI (they are git-ignored). Generate them once, then install:

```bash
cd mobile

# 1) Generate the native projects (name must be EnviroMasterBI)
npx @react-native-community/cli@0.74.5 init EnviroMasterBI --directory .tmp --skip-install
mv .tmp/android ./android
mv .tmp/ios ./ios
rm -rf .tmp

# 2) Install JS deps
npm install

# 3) Link vector-icon fonts (uses react-native.config.js)
npx react-native-asset

# 4) iOS pods
npm run pods            # cd ios && pod install
```

## Run
```bash
npm start                 # Metro
npm run ios               # or: npm run android
```

## Point the app at your backend
Edit `src/constants/env.js` (default `http://localhost:4000/api/v1`), or create
`src/constants/env.local.js` (git-ignored) from `env.local.example.js`:

```js
export default { apiBaseUrl: 'http://192.168.1.50:4000/api/v1', reportingTz: 'America/New_York' };
```
- **iOS simulator** — `localhost` works.
- **Android emulator** — use `http://10.0.2.2:4000/api/v1`.
- **Physical device** — use your computer's LAN IP; backend must be reachable on the same Wi‑Fi.

## Feature notes
- **Payroll CSV upload** is implemented natively: Payroll Cost → *Upload payroll CSV* opens the
  document picker and multipart-uploads to `POST /payroll/upload` (same endpoint as web).
- **Sync buttons** (Distances, Customers) trigger background sync and poll status, disabled while running.
- **Deep drill-down** on all Revenue screens: row → route → customers → items + invoices → invoice line items.
- Tables scroll horizontally; tap a row to drill in. Date filter includes **All time**; the selected
  range is shared across screens (default **This Year**).
